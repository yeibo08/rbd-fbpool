# Football pool

## What is a football pool

A football pool is a competitive framework where users predict match outcomes to earn points. This spec outlines a system for managing entries, scoring results, and tracking real-time leaderboards.

## Core Concepts

**User** Is a user of the system that has a unique and idempotent id. It has a profile with display name and is linked to a email and password. Could be added to a group of friends

**Group of friends** Group of max 20 users that share a leaderboard of points. They can also create a set of customized pointing rules

**Pointing rules** Set of customized rules to stack users from the group of friends to determine a winner. The rules could include match result, goals scored by each team and sum of goals for both teams.

**Fixture** The complete calendar of the tournament with all the matches. Specified in the fixture.md file

**Leaderboard** Table with the name and total accumulated points of each user within a group of friends

**Match** Match between two teams with the national flag, national team name, venue, date and time of the match. Prior the match users could set their score guess and after the match it should be compared with the official result.

## Winner

- The user with most points at the end of the tournament accordingly to the fixture is the winner

## Features

**Login** You must register with a valid email, create a password equal or greater than 8 characters, with at least 1 capital letter, at least 1 number and at least 1 of the following symbols !#$?. The users must be stored in a database (SQLite preferred) with encrypted password and unique idempotent identifier. In case user forgets the password, a new password will be generated by the system allowing them to change after the first login with the system generated password.

**User profile** Each user could customize they display name, they password and could visualize their groups of friends they are participating with. Each user could share his profile with a unique QR generated in the profile screen and this could be used by a group of friends admin to join them to the group.

**Group of friends** A user can create a group of friends and send invites with a specific link for the group. The group is restricted to a maxium of 20 friends. The group of friends should have an unique idempotent identifier. Each user could have 3 roles, owner will be the one that creates the group, manager that could create or modify the pointing rules and common user who just participates in the pool.

**Customized pointing rules** Owner and managers of the group of friends could establish customized pointing rules that will apply for each match. The rules will only include match result (winner/loser/draw), goals scored by each team and sum of goals scored by both teams. For playoff matches, even when the official result could be a draw after regular and overtime, consider winner the team that wins in penalty kicks. The penalty kicks goals are not considered for the goals scored by each team and the sum of both teams

**Match center** Each user could take a look of all the matches sorted by date, including upcoming matches and past matches. For the upcoming matches the user should introduce the guess of the score max 15 min prior the kickoff and could modify as much as they want before this window. User could check results from the past matches and see how many points they earn according to the pointing rules. Each group of friends should have their own match center but the interface must be the same for all the group of friends. Base this in the fixture.md doc
